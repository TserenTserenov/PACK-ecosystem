# Метод: Верификация коммуникации через банк персон-скептиков Лида

## 1. Определение

Метод адверсариальной проверки материалов Конвейера продвижения (посты, письма, слайды, 1:1-сообщения наставников), адресованных Лиду, до публикации. Прогоняет текст через именованные архетипы возражений реального Лида (портрет + скрытые допущения + триггер отторжения) — каждая персона либо убеждена, либо называет, что конкретно оттолкнёт.

**Не изобретён на Ф1** — формализует уже работающую практику: первый прогон (карта возражений марафона волны-2, WP-330, 6 агентов-персон) нашёл 40 реальных дефектов формулировок за одну сессию.

## 2. Параметр метода: банк персон

Банк персон-скептиков (`DS-ecosystem-development/A.Systems-Builder/A1.Teams-and-Communities/A1.1.Meaning/1.1.4. User-Research/Банк персон-скептиков/personas.md`) — **входной параметр этого метода**, не состояние `ECO.SYS.001-lead`. Уточнение зафиксировано в консенсусе пир-сессии `2026-07-10-32-wp251-ontology-types` (ход 1-2): портреты — таксономия входов, не онтология системы Лид.

**Постоянный дом банка** — `DS-ecosystem-development/.../1.1.4. User-Research/Банк персон-скептиков/` (перенос выполнен 2026-07-11, WP-251 Ф4.5; процессы использования — README.md рядом с банком). Банк — операционные данные (token-level), не Pack-контент: в PACK-ecosystem живёт только этот метод.

## 3. Второй параметр: тип коммуникации

Один метод с разными шаблонами прогона по параметру «тип коммуникации» (пост / письмо / 1:1-сообщение) — решение Ф0 (`ECO.SC.003`): дробление по каналам не оправдано.

## 4. Режим применения

`may` — рекомендуемый, не блокирующий гейт (см. `ECO.SC.003 §Контроль качества коммуникации`). Отказ от прогона не блокирует публикацию; находки метода — список рекомендаций автору, не контракт.

## 5. Точка приложения в ECO.SYS.001

Переход `холодный → прошёл первый контакт` (состояние 1→2) и переход `2 → покупатель` (`ECO.M.022`) — оба момента, где коммуникация с Лидом критична; метод применяется к материалу ДО отправки, не к самому переходу состояния.

## 6. Сценарии использования

См. `ECO.SC.003 §Сценарии использования` (Алёна перед публикацией / агент-автор контента / наставник Марафона 1:1) — идентичны для этого метода, `ECO.SC.003` уже их специфицировал на Ф0.

## 7. Не покрывает

- Наполнение и обновление самого банка персон (процесс описан в `personas.md §Как наполнять`) — операционная практика, не предмет этого Pack-метода.
- Институционализацию «где и когда обязателен прогон» — решена пилотом отдельно (WP-251.md: «перед публикацией ЛЮБОГО материала Конвейера продвижения»), не входит в определение метода.
