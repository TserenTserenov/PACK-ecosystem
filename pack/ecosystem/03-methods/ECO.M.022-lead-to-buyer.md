# Метод: Лид → Покупатель

## 1. Определение

Метод перевода Лида (`ECO.SYS.001`, состояние 2 «прошёл первый контакт/диагностику») в граничное состояние «покупатель» — точку выхода из системы Лид, описанную Алёной (доклад 19 апр 2026, WP-233) как «методы донесения + изменения представления о мире/себе». Момент, когда Лид принимает решение о покупке/вступлении.

## 2. Вход/выход

- **Вход:** Лид в состоянии 2 (прошёл первый контакт), опционально прошедший контроль качества коммуникации (`ECO.M.024-lead-persona-verification`, `may`).
- **Выход:** экземпляр покидает `ECO.SYS.001`, попадает в граничное состояние «покупатель» (не отдельная Pack-система на Ф1 — см. `ECO.SYS.001-lead.md §Boundary states`).

## 3. Токен-уровневый пример (WP-385, уровень 1)

Марафон систематичности — первый платный шаг. Согласно WP-385: «запускает изменение представления о мире/себе (Алёна, система 1)». Composite-триггер автоматики онбординг-серий WP-385 — конкретный механизм этого метода для домена «Личное развитие».

## 4. Не покрывает

- Точный механизм принятия решения (маркетинговая воронка, психология выбора) — за пределами Pack-типизации, домен операционного содержания Алёны.
- Дальнейший переход «покупатель → член сообщества» — отдельный метод `ECO.M.023-buyer-to-member`.
- Мост к БД (таблица подписок/платежей, если появится) — Ф2 WP-251, dep WP-228.

## 5. Открытые вопросы

Если на Ф2 (гэп-матрица) обнаружится, что «покупатель» требует собственного Pack-типа (`ECO.SYS.004-buyer`) — этот метод получит уточнённый выход (не в «безымянную границу», а в новую систему). Решение по пир-сессии `2026-07-10-32-wp251-ontology-types`.
