# ECO.M.027 — Benchmark Showcase: бенчмарк против одного известного конкурента

## Определение

Метод проектирования публичной витрины продукта: выбрать одного хорошо известного конкурента («чемпиона ниши») и строить showcase как ответ на вопрос «чем это лучше X?» — вместо абстрактного представления всей ниши.

## Forces

_(Optional, WP-448 Ф12) Какие конкурирующие давления удерживает метод._

| Force | Tension |
|-------|---------|
| Узнаваемость эталона ↔ честность контраста | §«Правило выбора эталона» п.1 и п.2: эталон должен быть узнаваем аудиторией без объяснений, но одновременно иметь «реальные сильные точки» — чем сильнее выбранный конкурент, тем труднее выигрышно с ним контрастировать |
| Один эталон ↔ несколько сегментов аудитории | §«Ограничения»: «один эталон = один сегмент аудитории»; концентрация на одном узнаваемом конкуренте (сила метода) конкурирует с охватом нескольких сегментов, каждый из которых может знать разных «чемпионов ниши» |
| Контрастный нарратив ↔ данные, подтверждающие превосходство | §«Ограничения»: метод требует «собственных данных для подтверждения преимущества» (иначе неотличимо от вендорского нарратива, ECO.FM.003), но сила подхода — именно в простом контрастном сообщении «чем лучше X» |

## IPO

- **Вход:** продукт/платформа, выходящая в нишу с устоявшимися альтернативами; целевая аудитория
- **Процесс:** выбрать одного узнаваемого конкурента с сильными точками → сформулировать контрастный ответ → построить showcase вокруг контраста
- **Выход:** витрина, отвечающая на «чем лучше X?» для конкретного X

## Правило выбора эталона

Эталон (X) должен:
1. **Быть узнаваем** целевой аудиторией без объяснений
2. **Иметь реальные сильные точки** — тогда контраст читается как честный
3. **Занимать ту же нишу** — аудитория видела X, слышала о нём, может сравнить

Неправильный эталон: абстрактный «рынок AI-инструментов» или «другие решения» — аудитория не знает, с чем сравнивать.

## Пример применения (WP-401)

IWE showcase: вместо «IWE — платформа для работы с ИИ-агентами» → «IWE vs claude-obsidian: чем IWE лучше для управления знаниями с ИИ?» claude-obsidian узнаваем целевой аудиторией, имеет сильные точки — контраст читается.

## Применимость

Любой продукт, выходящий в нишу с устоявшимися известными альтернативами. Тест: «Есть ли у ниши "чемпион", которого знает моя аудитория?» Да → применять этот метод.

## Bias-Annotation

_(Optional, WP-448 Ф12) Куда систематически съезжает внимание практикующего._

| Bias | Direction of distortion |
|------|--------------------------|
| Узнаваемость эталона vs собственные данные | Внимание автора витрины съезжает к подбору хорошо узнаваемого конкурента (§«Правило выбора эталона») и уходит от требования §«Ограничения» — подтвердить контраст собственным датасетом, без которого заявление «неотличимо от вендорского нарратива» (ECO.FM.003) |
| Один найденный «чемпион ниши» vs множественность сегментов | Внимание смещается на успешный прецедент (§«Пример применения», IWE vs claude-obsidian) и недооценивается оговорка §«Ограничения»: при нескольких сегментах аудитории один эталон может не работать для всех |

## Ограничения

- Требует собственных данных для подтверждения преимущества (ECO.FM.003: заявление без датасета = неотличимо от вендорского нарратива)
- Один эталон = один сегмент аудитории; при нескольких сегментах может потребоваться несколько витрин с разными эталонами
