# ECO.M.017: Ecosystem-Led Growth (ELG)

> **Назначение:** Метод роста, при котором партнёры-организации (не пользователи-индивиды) становятся главным каналом привлечения, конверсии и удержания.

---

## 1. Определение

ELG — стратегия, при которой экосистема партнёров (технологические интеграции, реселлеры, со-маркетинг) = первичный growth engine. Данные и отношения из партнёрской сети используются для ускорения каждого этапа воронки.

**Автор:** Bob Moore (Co-Founder Crossbeam). Книга: *«Ecosystem-Led Growth: A Blueprint for Sales and Marketing Success Using the Power of Partnerships»* (Wiley, 2024, ISBN 9781394226832).

**Манифест:** [ELG Alliance](https://elgalliance.com/)

---

## 2. ELG vs PLG vs CLG vs SLG

| | PLG | CLG | ELG | SLG |
|---|---|---|---|---|
| **Двигатель роста** | Продукт (self-serve) | Сообщество пользователей | Партнёры-организации | Sales team |
| **Acquisition** | Free trial / freemium | Word-of-mouth, peer content | Partner referrals, co-marketing | Outbound, demos |
| **Conversion** | In-product upgrade | Trust from community | Warm intros, shared data | Sales cycle |
| **Retention** | Product value | Social accountability | Integration depth (switching cost) | CSM relationship |
| **Метрики** | Activation rate, PQL | Community engagement, NPS | Partner-sourced pipeline, overlap % | SQL, ACV |
| **Когда работает** | B2C, low ACV | Expert products, B2C/B2B | Multi-product ecosystems, B2B | Enterprise, high ACV |

**Ключевое различение:** CLG = рост через сообщество *людей*. ELG = рост через партнёрства *организаций*. Разные механики, разные метрики, разные инвестиции.

---

## 3. Три столпа ELG

### 3.1 Ecosystem Data

Общие данные между партнёрами: overlap accounts, shared opportunities, customer signals.

**Инструменты:** Crossbeam, Reveal — платформы для безопасного обмена данными о пересечении клиентских баз.

### 3.2 Ecosystem Playbooks

Операционализация партнёрства: co-sell (совместная продажа), co-market (совместный маркетинг), co-build (совместная интеграция).

| Playbook | Механизм | Пример |
|----------|---------|---------|
| **Co-sell** | Партнёр рекомендует вас своим клиентам | LMS-вендор рекомендует AI coaching |
| **Co-market** | Совместный контент, вебинары, case studies | Webinar «LMS + AI: как повысить retention» |
| **Co-build** | Совместная интеграция, API, plugin | Digital Twin виджет внутри LMS |

### 3.3 Ecosystem Ops

Команда/функция, управляющая партнёрским pipeline: tracking, attribution, activation.

## 4. Метрики ELG

| Метрика | Формула | Benchmark |
|---------|---------|-----------|
| Partner-sourced pipeline | Deals from partner referrals / Total pipeline | >30% (mature ELG) |
| Partner-influenced revenue | Revenue where partner involved / Total | >50% (Salesforce: 70%) |
| Account overlap | Shared accounts / Total accounts | Depends on ecosystem |
| Integration adoption | Users with 5+ integrations / Total | 7-10× retention (Salesforce data) |
| Partner activation rate | Active partners / Total partners | >40% |

**SOTA 2026:** IDC прогнозирует 60% глобальной выручки через партнёрские модели к 2026.

---

## 5. ELG Flywheel

```
Интеграция → Adoption → Data overlap → Co-sell opportunity →
  → Revenue → Deeper integration → More adoption → ...
```

**Switching cost:** Чем глубже интеграция партнёра, тем выше retention. Shopify: 32% новых мерчантов через партнёрские приложения (2025). Salesforce: 7-10× retention при 5+ installed apps.

## 6. Применение к IWE

| Аспект | Текущее | ELG-потенциал |
|--------|---------|---------------|
| **Партнёры-интеграции** | Нет формальных | LMS providers, AI tool vendors, HR/L&D platforms |
| **Co-sell** | Нет | Менторы/выпускники рекомендуют экосистему своим клиентам |
| **Co-market** | Founder's personal brand | Совместные вебинары с LMS/AI partners |
| **Co-build** | Bot + LMS (внутренняя) | Agent SDK → third-party agents = ecosystem |
| **Data overlap** | Нет | Shared learning outcomes с B2B-клиентами |

### Когда включать ELG для IWE

- **Сейчас (seed):** PLG + CLG. ELG преждевременен.
- **После PMF:** Первые 2-3 формальных партнёрства (co-market).
- **Series A:** ELG ops function, Agent SDK как ecosystem enabler.

## 7. Failure modes

| ID | Ошибка | Последствие |
|----|--------|-------------|
| FM.ELG.001 | Начать ELG до PLG/CLG | Нет собственной ценности → нечего предложить партнёрам |
| FM.ELG.002 | Партнёрства без data sharing | Нет visibility → нет attribution → нет ROI |
| FM.ELG.003 | Слишком много партнёров сразу | Thin spread, нет глубины ни с кем |
| FM.ELG.004 | Co-sell без co-build | Рекомендация без интеграции → low conversion |
| FM.ELG.005 | Ignore partner activation | 80% партнёров inactive → vanity metric |
