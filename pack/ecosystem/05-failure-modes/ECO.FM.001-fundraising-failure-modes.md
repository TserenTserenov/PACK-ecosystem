# ECO.FM.001: Типовые ошибки привлечения инвестиций

> **Назначение:** Реестр failure modes fundraising-процесса. Извлечены из ECO.M.001 и дополнены.
> **Использование:** Проверять перед каждым шагом fundraising — какие ошибки характерны для текущей стадии.

---

## Pitch & Presentation

| ID | Ошибка | Последствие | Как избежать |
|----|--------|-------------|-------------|
| FM.FR.001 | Завышенная оценка (pre-money) | Down round позже → потеря доверия | Benchmarks по стадии и отрасли |
| FM.FR.002 | Нет clear ask (сумма + на что) | Инвестор не понимает что финансирует | Таблица: направление → сумма → результат |
| FM.FR.003 | Traction gap (мало данных) | «Приходите когда подрастёте» | Показать trajectory, не snapshot |
| FM.FR.004 | Слабая команда (нет бизнес-лидера) | «Кто будет продавать?» | Hiring plan или advisory |
| FM.FR.005 | Template-following pitch | Теряется impressive-first, инвестор скучает | YC First Minute Rule: lead with strongest |
| FM.FR.006 | Complexity flex (jargon) | Investor отключается в первую минуту | 2 предложения + 1 пример = clarity |
| FM.FR.007 | Traction без timeframe | «10K users» без контекста = ничего | Momentum = traction / time |
| FM.FR.010 | Vanity metrics вместо revenue | «10M users» без revenue per user | Unit economics: ARPU, LTV, CAC |

## Financial Projections

| ID | Ошибка | Последствие | Как избежать |
|----|--------|-------------|-------------|
| FM.FR.008 | Top-down only | «$100B market × 1%» → мгновенный отказ | Bottom-up: segment × conversion × ARPU |
| FM.FR.009 | Single-line forecast | «Будем расти 10x/год» без cohort breakdown | 12-24 cohorts, retention curve |
| FM.FR.011 | Нет sensitivity analysis | Единственный best-case сценарий → недоверие | Base / optimistic / pessimistic |

## Due Diligence

| ID | Ошибка | Последствие | Как избежать |
|----|--------|-------------|-------------|
| FM.FR.012 | Нет data room до первой встречи | Задержка 2-3 недели → потеря momentum | Подготовить data room заранее |
| FM.FR.013 | Cap table с ошибками | Red flag → углублённая проверка → отказ | Ревизия cap table перед fundraising |
| FM.FR.014 | IP не оформлен на компанию | Стоп-фактор. Инвестор не закроет сделку | IP assignment до начала raise |

## Outreach & Pipeline

| ID | Ошибка | Последствие | Как избежать |
|----|--------|-------------|-------------|
| FM.FR.015 | Массовая рассылка без персонализации | Spam-репутация, 0% конверсия | 3 предложения: что + traction + ask |
| FM.FR.016 | Нет warm intro strategy | Cold-only → 3-5x ниже конверсия | Mapping: кто знает инвестора? |
| FM.FR.017 | Pipeline <50 контактов | Математически недостаточно для раунда | Target list 100-200 инвесторов |
| FM.FR.018 | Нет follow-up в 24ч | Investor interest decays exponentially | Автоматическое напоминание в CRM |

## Structural

| ID | Ошибка | Последствие | Как избежать |
|----|--------|-------------|-------------|
| FM.FR.019 | SAFE stacking без учёта dilution | Невидимое размывание → шок на Series A | Cap table simulator перед каждым SAFE |
| FM.FR.020 | Nет competitor slide | «Наивные основатели» | Честная матрица: мы vs альтернативы |
