# ECO.D.005: Platform ≠ Product ≠ Ecosystem

## Различение

| | Product | Platform | Ecosystem |
|---|---|---|---|
| **Что** | Решение конкретной проблемы | Инфраструктура для создания решений | Система взаимодействующих акторов |
| **Ценность создаёт** | Команда продукта | Команда + third parties через API | Все участники друг для друга |
| **Масштабирование** | Линейное (больше фич → больше людей) | Субконтрактное (third parties расширяют) | Сетевые эффекты (каждый +1 → ценность для всех) |
| **Конкурентное преимущество** | Фичи, UX, цена | Размер сети разработчиков | Плотность взаимодействий |
| **Стратегический вопрос** | Что строим? | Кого пускаем строить? | Как создать ценность между участниками? |
| **Валюация (SOTA 2025)** | 1× | 2-3× | 3-5× |
| **Пример** | Zoom (видеозвонки) | Shopify (магазин + apps) | Apple (devices + App Store + developers + users) |

## Почему важно различать

**Путаница Product → Platform:** Продукт с API — ещё не платформа. Платформа возникает когда третья сторона может создавать ценность через ваш API *для ваших пользователей*. API для внутренней интеграции — это инженерная практика, не платформенная стратегия.

**Путаница Platform → Ecosystem:** Платформа с приложениями — ещё не экосистема. Экосистема возникает когда участники взаимодействуют *друг с другом* без посредничества платформы. Marketplace apps — это платформа. Сообщество, где пользователи помогают друг другу и создают контент — это экосистема.

**Путаница Ecosystem → Product:** Экосистема без ядра продукта — набор несвязанных сервисов. Без собственного value proposition нечего масштабировать.

## Стратегическая прогрессия

```
Product → Platform → Ecosystem
```

Это не линейный путь, а **расширение**: Product остаётся ядром, Platform добавляет точки расширения, Ecosystem добавляет взаимодействия между участниками.

**Критическая ошибка:** Пытаться построить Ecosystem, не имея работающего Product. Сетевые эффекты усиливают то, что уже есть — если ядро слабое, усиливать нечего.

## Пример (IWE)

| Уровень | Что это в IWE |
|---------|---------------|
| **Product** | Курсы и методология системного мышления |
| **Platform** | LMS + Digital Twin + Agent SDK (третьи стороны создают агентов) |
| **Ecosystem** | Методология + AI-агенты + Сообщество + Партнёры-менторы + Выпускники-как-авторы |

## Failure modes

| ID | Ошибка | Последствие |
|----|--------|-------------|
| FM.PPE.001 | Называть продукт «экосистемой» для маркетинга | Завышенные ожидания инвесторов, отсутствие реальных network effects |
| FM.PPE.002 | Строить платформу до product-market fit | Инвестиции в расширяемость при нуле пользователей |
| FM.PPE.003 | Закрывать платформу, убивая экосистему | Потеря third-party ценности (Twitter API 2023) |
