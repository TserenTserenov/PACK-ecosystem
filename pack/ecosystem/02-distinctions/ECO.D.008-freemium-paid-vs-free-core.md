# ECO.D.008: Freemium с бесплатным core ≠ Freemium с платным внешним core

## Различение

Бизнес-модель «freemium» структурно работает по-разному в зависимости от того, бесплатен ли core-ресурс для провайдера:

| Класс | core-ресурс | Маржинальная стоимость одного free-пользователя | Возможна ли «бесплатный полный доступ»? |
|-------|-------------|------------------------------------------------|----------------------------------------|
| **classic SaaS** (Slack, Notion, Figma free) | свой стек / БД-вызовы | пренебрежимо мала | Да — «trial 15 дней FREE полного доступа» работает |
| **AI-обёртки на платном LLM** (IWE, обёртки на Claude/GPT, transcription-сервисы, embedding-search) | внешний LLM/API провайдер, каждый вызов оплачен | $X/месяц на пользователя, X>0 | **Нет** — без покрытия COGS = bleed |

## Тест

«Бесплатный пользователь обходится мне в $0/месяц или в $X/месяц?»

- $0 → freemium возможен по времени (trial 15 дней), модель Red Hat работает.
- $X>0 → freemium требует **структурного ограничения**, не временного.

## Реализуемые варианты T1 для класса «AI-обёртка на платном LLM»

- (а) **Ограниченная квота LLM** (Discovery-tier): N запросов в месяц, дальше — оплата.
- (б) **BYOK** (Bring Your Own Key): пользователь приносит свой API-ключ к LLM-провайдеру, COGS для платформы = 0.
- (в) **Demo-период** с фиксированным cap (расходы покрыты бюджетом маркетинга).

**Уточнение про BYOK:** НЕ решает retention-задачу — пользователь может уйти к провайдеру напрямую. Нужны другие моаты (Pack-контент, ритуалы, среда сопроизводителей).

## Контраст с classic SaaS

В classic SaaS-продуктах «trial 15 дней FREE полного доступа» работает, потому что DB-вызовы и compute практически бесплатны для провайдера. Применять ту же модель к AI-обёртке без COGS-анализа — типовая ошибка ранних стартапов.

## Применимость

Все SaaS-продукты, у которых core-вызовы оплачиваются третьему провайдеру:
- LLM-обёртки (Claude, GPT, Anthropic API клиенты),
- AI-агенты на внешних моделях,
- video-generation на API,
- transcription / embedding-search.
