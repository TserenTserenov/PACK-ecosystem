# Core Distinctions: Экосистема развития интеллекта

## Обязательные различения

### D.ECO.001: Экосистема ≠ Набор ресурсов

| Экосистема | Набор ресурсов |
|------------|----------------|
| Связанная целостность | Коллекция элементов |
| Бесшовные переходы | Независимые единицы |
| Общая цель | Разрозненные функции |

**Почему важно**: Набор ресурсов без связей не создаёт экосистему. Экосистема предполагает спроектированные переходы и общую архитектуру.

---

### D.ECO.002: Компонент ≠ Продукт

| Компонент экосистемы | Продукт |
|----------------------|---------|
| Функция в системе | Самостоятельная ценность |
| Связан с другими | Автономен |
| Оценивается по вкладу в целое | Оценивается сам по себе |

**Почему важно**: Компонент экосистемы имеет смысл в контексте целого. Курс как продукт и курс как компонент экосистемы — разные вещи.

---

### D.ECO.003: Бесшовность ≠ Единообразие

| Бесшовность | Единообразие |
|-------------|--------------|
| Отсутствие барьеров | Одинаковый формат |
| Плавные переходы | Один стиль везде |
| Связность опыта | Унификация |

**Почему важно**: Бесшовность достигается не унификацией, а качеством переходов между разными компонентами.

---

### D.ECO.004: Путь ≠ Маршрут обучения

| Путь в экосистеме | Маршрут обучения |
|-------------------|------------------|
| Траектория через компоненты | Последовательность уроков |
| Может быть нелинейным | Линейная структура |
| Определяется состоянием | Определяется программой |

**Почему важно**: Путь в экосистеме адаптивен к состоянию созидателя, а не жёстко предписан.

---

### D.ECO.005: Точка входа ≠ Начальный курс

| Точка входа | Начальный курс |
|-------------|----------------|
| Любой первый контакт | Обучающий продукт |
| Может быть событием, сообществом | Контент |
| Зависит от созидателя | Фиксирован |

**Почему важно**: В экосистеме много точек входа для разных типов созидателей.

---

### D.ECO.006: Качество экосистемы ≠ Качество компонентов

| Качество экосистемы | Качество компонентов |
|---------------------|----------------------|
| Бесшовность целого | Качество частей |
| Связность | Контент |
| Конверсия переходов | Удовлетворённость |

**Почему важно**: Отличные компоненты могут образовывать плохую экосистему, если между ними барьеры.

### D.ECO.007: Обучение человека ≠ Обучение ИИ (но параллели)

*Origin: EDU.D.011 | s2r: F5*

| Человек | ↔ Параллель | ИИ |
|---------|-------------|-----|
| Кривая забывания (Ebbinghaus) | Потеря информации | Catastrophic forgetting |
| Рабочая память (7±2) | Ограничение обработки | Context window |
| Desirable difficulties (Bjork) | Оптимальная трудность | Regularization, dropout |
| Scaffolding (ZPD) | Постепенное усложнение | Curriculum learning |
| Экзокортекс (внешняя память) | Расширение ресурсов | RAG |
| Spaced repetition | Интервальное повторение | Rehearsal methods |

**Почему важно**: Параллели позволяют (1) проектировать ИИ-ассистентов с учётом когнитивных ограничений, (2) использовать метафоры ИИ для объяснения обучения, (3) находить новые методы по аналогии. Но различия ФУНДАМЕНТАЛЬНЫ — сознание, мотивация, эмоции, телесный опыт.

**Связи:** MIM.FORM.004 (Three-Level Method Hierarchy — мета-принципы универсальны), DP.CONCEPT.001 (Цифровой двойник).

**SoTA:** emerging. Zador (2019), Lake et al. (2017), Bengio (2017).

## Применение различений

При описании экосистемы проверяйте:
- [ ] Не путаете ли экосистему с набором ресурсов?
- [ ] Не путаете ли компонент с продуктом?
- [ ] Не путаете ли бесшовность с единообразием?
- [ ] Различаете ли путь и маршрут?
- [ ] Не переносите ли НАПРЯМУЮ аналогии человек ↔ ИИ?

## ECO.D.019 — Платформенный трекер активности ≠ Прокси-измеритель

**Источник:** peer-сессия 2026-05-26-16-wp337-parliament-boundary (report.md, тема 4).

**Различение.** WakaTime / GitHub commits / IDE session time измеряют общее время активности в репо, не инвестиции в конкретный домен:
- Трекер считает все коммиты, включая рутинные (merge, update, fix конфликтов)
- Без фильтра по scope (репо / домен / тип задачи) число завышается в 3–5×
- При использовании как прокси «инвестиций в X» — систематическое искажение

**Критерий:**
- Если метрика используется для измерения инвестиций в конкретный домен (learning, написание Pack, проект) — нужен domain-scoped фильтр
- Если метрика используется как общий индикатор активности (есть/нет работа) — трекер достаточен

**Применимо:** к любым quantitative self-системам, где нужно измерить не «сколько работал», а «сколько инвестировал в Y».

**Связи:**
- DP.FM.036 WakaTime Measurement Scope Bias (FM для этого же паттерна, в DP Pack)

## ECO.D.020 — Разрыв подачи: проданный результат ≠ публично доставляемое условие

**Источник:** session-close 2026-07-10 (марафон, WP-330); extraction-report `2026-07-10-inbox-check-3.md` #4.

**Различение.** Программа может **продавать результат** (рабочие продукты, план на месяц), а **публично доставлять условие** (бытовая привычка — «стакан воды по утрам»). Возражение участника «слишком мелко» — не каприз и не признак слабого оффера, а симптом **непоказанного моста**: связь между условием и проданным результатом не явлена публично.

**Критерий:**
- Если возражение направлено на «мелкость» публичного действия — ищи разрыв подачи, а не дефект продукта
- Лечение — показать мост (как условие превращается в результат), а не укрупнять условие или менять результат

**Применимо:** к публичным материалам любой программы экосистемы, где входное действие выглядит бытовым, а обещание — результатным (марафоны, треки привычек, онбординг).

**Связи:**
- RHE.METHOD.008 (PACK-rhetoric) — карта возражений через скептиков: «слишком мелко» — один из типов возражений, который этот метод ловит до живой встречи
